# 범용 디자인 Craft 규칙

어떤 디자인 스킬(슬라이드, 웹, 대시보드, 카드 등)에서도 적용되는 공통 규칙.  
스킬별 특수 규칙은 각 스킬 파일이 담당하고, 이 파일은 공통 베이스라인을 제공한다.

---

## 1. Anti-AI-Slop — AI 생성 결과물 식별 패턴 차단

### P0 — 반드시 제거 (craft 실패)

1. **발명된 수치** — "10배 향상", "99% 정확도", "3배 효율", "업무 효율 극대화 30%" 등. 원본 데이터에 없는 수치는 사용 금지. 수치가 없으면 수치 없이 설명한다.
2. **AI 클리셰 문구** — 혁신적인·원활한·극대화·시너지·솔루션·최적화·스마트한·강력한·게임체인저·원스톱. 구체 사실로 대체한다.
3. **플레이스홀더 콘텐츠** — "lorem ipsum", "항목1/항목2/항목3", "추가 예정", 빈 카드, 빈 섹션. 내용이 없으면 그 섹션/슬라이드를 제거한다.
4. **이모지를 기능 아이콘으로 사용** — ✨🚀🎯⚡🔥💡를 헤더·버튼·리스트 아이콘으로. SVG 아이콘 또는 텍스트 마커(■ ▶ •)로 대체한다.
5. **신뢰 그라디언트** — 보라→파랑, 파랑→청록 등 두 색 그라디언트를 주요 배경·헤더에. 플랫 서피스 + 타이포 위계로 대체한다.
6. **색만으로 의미 전달** — 빨간색=오류라는 것을 색만으로 표현. 아이콘·레이블·형태도 함께 차별화한다.

### P1 — 가능하면 제거 (soft tell)

- 표준 섹션 순서를 변형 없이 그대로 사용 (예: 헤더→기능→가격→FAQ→CTA)
- 대칭·균일 레이아웃이 전부 — 의도적인 비대칭 또는 밀도 변화 없음
- 장식용 blob/wave SVG 배경 — 내용을 위한 형태만 허용

## 2. 타이포그래피 자간 (Letter-spacing) — 가장 많이 놓치는 규칙

AI 생성 디자인을 구분하는 가장 신뢰도 높은 지표.

| 컨텍스트 | 자간 |
|---------|------|
| 본문 (14~18pt) | 0 (기본값) |
| 소형 텍스트 (11~13pt) | +0.01em ~ +0.02em |
| UI 라벨·버튼 텍스트 | +0.02em |
| **ALL CAPS (모든 크기)** | **+0.06em ~ +0.1em (필수)** |
| 헤딩 32pt 이상 | -0.01em ~ -0.02em |
| 디스플레이 48pt 이상 | -0.02em ~ -0.03em |

**ALL CAPS 자간 없음 = 글자가 붙어 보임 = craft 실패 1위**  
**대형 헤딩 자간 없음 = 글자가 느슨해 보임 = craft 실패 2위**

### 3가지 굵기 체계

대부분의 잘 만들어진 UI는 굵기를 3가지만 사용한다:
- **읽기용** (400) — 본문
- **강조용** (510~550) — UI 텍스트, 라벨, 네비게이션
- **공표용** (590~600) — 헤드라인, 버튼

700+ 굵기는 드물게 필요하다. 모든 곳에 Bold를 쓰면 위계가 없어진다.

## 3. 색상 규율 — Accent 사용 제한

### 팔레트 구조

| 레이어 | 전체 면적 비율 | 역할 |
|--------|-------------|------|
| 중립색 (배경, 서피스, 텍스트) | 70~90% | 공간 대부분 |
| Accent (브랜드 색, 1가지만) | 5~10% | 핵심 강조 1~2개 |
| 시맨틱 (성공, 경고, 오류) | 0~5% | 상태 표시 전용 |
| 효과 (그라디언트, 글로우) | <1% | 극히 드물게 |

### Accent 사용 상한

- **화면(슬라이드)당 accent 가시적 사용 최대 2회**
- 전형적인 쌍: eyebrow 라벨(1) + 강조 카드 또는 CTA(1)
- 링크, hover 링 등도 accent 사용으로 카운트
- 3회 이상이면 accent가 희석되어 강조 효과가 사라진다

### 다크 테마 배경 기준

- 배경: `#0f0f0f` 이상 (순수 `#000` 금지 — 진동 발생)
- 전경: `#f0f0f0` 이하 (순수 `#fff` 금지)
- 다크 서피스 border: `rgba(255,255,255,0.08)` 반투명 흰색 권장

## 4. 80/20 소울 원칙

**80% 검증된 패턴 + 20% 이 작업만의 차별점 1가지.**

20%의 차별점 후보:
- 하나의 강렬한 시각적 선택 — 타이포그래피 결정, 단일 색상 결정, 예상치 못한 비율
- 마이크로카피 — "시작하기" 대신 "추적 시작"처럼 제품을 써본 사람이 쓸 수 있는 문구
- 도메인 지식을 증명하는 디테일 — 제품 특유의 상태 배지, 실제 수치, 내부 용어를 풀어 쓴 설명

**완성 기준**: 이 결과물의 스크린샷을 같은 조직 사람이 보면 어떤 프로젝트인지 알 수 있어야 완성이다. 아무 회사에나 붙여넣어도 맞으면 템플릿이지 작업물이 아니다.

## 5. 콘텐츠 무결성

- **원본에 없는 내용 추가 금지** — 수치, 고유명사, 시스템명은 원본 그대로. 새로운 사실 추가는 절대 금지.
- **내용 없음 = 섹션 제거** — 채울 내용이 없을 때 플레이스홀더로 채우지 않는다. 해당 섹션·슬라이드·카드를 삭제한다.
- **수치 먼저** — 수치가 있으면 수치를 앞에, 설명을 뒤에. "효율이 개선되었습니다 (30%)" 아니라 "30% 효율 개선".

## 사용 방법

다른 디자인 스킬의 `SKILL.md`에서 참조:

```
> 공통 디자인 craft 규칙: `~/.claude/skills/craft-design-rules.md`
> 이 파일의 규칙은 모든 디자인 작업에 적용된다.
```

또는 스킬 내부에서 특정 섹션만 인용:

```
# Anti-AI-Slop P0 체크리스트
→ craft-design-rules.md § 1. Anti-AI-Slop 참조
```
