# Spigen Slides Review Checklist

PPT 제작 완료 후 완료 보고 전 반드시 실행한다.  
`spigen_verify.py` PASS + 기획자 PASS + 디자이너(hard QA) PASS → 그때만 "완료"라고 말한다.  
대상 검수는 PASS/FAIL이 아니라 피드백 수집용이다.

---

## 자동 검증 — API로 슬라이드 직접 확인

각 항목을 Google Slides API로 읽어 판정한다.

- [ ] **폰트**: 한글이 포함된 텍스트는 `Noto Sans`, 그 외 텍스트는 `Proxima Nova`만 사용됐는가?
- [ ] **표지 구조**: 첫 슬라이드는 템플릿 cover 또는 `mk_cover()` 구조를 사용했는가?
- [ ] **off-canvas**: shape / text / divider / note 포함 모든 요소가 720×405 안에 있는가?
  - 확인: 모든 슬라이드의 `textElements[].textRun.style.fontFamily` 집계
  - FAIL 기준: 두 폰트 이외의 fontFamily가 존재하면 FAIL

- [ ] **오렌지 사용량**: `#FF6B1A` 계열 색상이 슬라이드당 3개 이하 요소에만 적용됐는가?
  - 확인: 슬라이드별로 `rgbColor`가 `{red:1.0, green:0.42, blue:0.10}` 근접인 요소 카운트
  - FAIL 기준: 한 슬라이드에서 4개 이상이면 FAIL

- [ ] **캔버스 bounds**: 모든 요소가 720×405pt 캔버스 안에 있는가?
  - 확인: `transform`(translateX/Y) + `size`(width/height)로 우/하단 끝점 계산
  - FAIL 기준: 오른쪽 끝 > 720 또는 아래쪽 끝 > 405이면 FAIL

## 수동 검증 — Claude가 슬라이드 내용 검토 후 직접 판단

우선순위는 아래 순서로 고정한다.

1. 내용 왜곡 없음
2. 실행/운영 전달 가능
3. 규격 안전
4. 미감/리듬은 그 다음

즉 "조금 심심하다"는 FAIL 사유가 아니고,
"내용이 잘렸다 / 실행 방법이 안 보인다 / 형식 때문에 설명이 사라졌다"가 FAIL 사유다.

내부 기준 파라미터 해석:
- `detail_level`이 높을수록 설명이 많은 것이 정상이다.
- `layout_variance`가 낮으면 안정적인 보고서형 배치를 우선한다.
- 따라서 "정보가 많다" 자체는 FAIL이 아니고,
  "정보가 있는데도 실행 방법이 안 보인다"가 FAIL이다.

슬라이드별 텍스트 내용과 구조를 API로 읽어 아래 기준으로 판단한다.

- [ ] **내용 보존**: 형식 때문에 원래 있어야 할 조건, 상태, 주체, 다음 액션이 빠지지 않았는가?
- [ ] **실행 가능성**: 운영용/디테일용 기준으로 이 슬라이드 한 장만 읽어도 해야 할 일이 보이는가?
- [ ] **메시지 집중**: 각 슬라이드에 경쟁하는 블록 없이 하나의 지배적인 메시지가 있는가?
- [ ] **스캔 가능성**: 각 슬라이드의 핵심 포인트를 3~5초 안에 파악할 수 있는가?
- [ ] **군더더기 없음**: 제거해도 의미가 손상되지 않는 카피·배지·콜아웃이 없는가?
- [ ] **섹션 구분 간결**: 섹션 구분 슬라이드가 포스터처럼 간결한가? (본문 설명 없이 번호+제목만)
- [ ] **테마 일관성**: 덱 전체가 단일 테마(dark 또는 light)로 통일됐는가? (dark/light 혼합 금지)
- [ ] **테마 타입**: 슬라이드 유형에 맞는 테마가 적용됐는가? (cover·section-divider → dark 고정, content → Q7 선택 테마)

### impeccable 방식 critique 항목 (참고용)

`impeccable`는 이 단계에서만 사용한다.  
즉 결과물을 보고 아래 항목을 critique / polish 관점으로 판단한다.

- [ ] **Visual Hierarchy**: 제목, 핵심값, 보조설명이 한눈에 우선순위가 구분되는가?
- [ ] **Pattern Repetition**: 같은 카드 구조가 의미 없이 반복되어 AI처럼 보이지 않는가?
- [ ] **Spacing Rhythm**: 모든 박스가 같은 padding으로 찍힌 느낌 없이 리듬이 있는가?
- [ ] **Density Fit**: 5~6줄 허용, 수치·근거 생략 없음, 자기완결 기준을 충족하는가?
- [ ] **Blandness Check**: 너무 안전하고 밋밋해서 “AI가 무난하게 만든 슬라이드”처럼 보이지 않는가?
- [ ] **Polish Check**: 정렬, 간격, 텍스트 줄 수, 강조 개수에서 어색한 지점이 없는가?

주의:
- 위 항목은 개선 힌트로 유용하지만, 운영용/보고서형 덱에서는 내용 전달보다 앞서지 않는다.
- 반복감이 조금 있더라도 내용 보존과 실행 가능성이 확보되면 완료 차단 사유가 아니다.

## 판정 및 처리

| 판정 | 조건 | 처리 |
|-----|-----|-----|
| ✅ 완료 가능 | 자동 + 수동 전 항목 PASS | "완료" 보고 및 링크 공유 |
| ❌ 완료 불가 | 하나라도 FAIL | FAIL 항목 명시 → 수정 → spigen_verify.py부터 재실행 |

FAIL 항목이 있을 경우 아래 형식으로 보고한다:

```
❌ 체크리스트 불통과

FAIL 항목:
- [폰트] 표지에 Gothic A1 사용 → Noto Sans로 교체 필요
- [메시지 집중] 슬라이드 2에 설명 블록 2개 경쟁 중

수정 후 재검수합니다.
```
